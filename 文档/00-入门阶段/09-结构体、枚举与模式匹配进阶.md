# 结构体、枚举与模式匹配进阶

## 1. 这一章是干什么的

前面你已经接触过：

- 变量
- 元组
- 控制流
- `match`

这一章开始把这些东西组合起来，进入 Rust 里非常重要的一组能力：

- 用 `struct` 描述有名字的复杂数据
- 用 `enum` 描述“一个值可能是几种形态之一”
- 用模式匹配把这些结构拆开

如果你后面想真正理解 `Option`、`Result`、异步状态、消息类型、领域模型，这一章是关键地基。

## 2. 学这一章前，你需要知道什么

你最好已经理解：

- 元组是什么
- `match` 的基础写法
- 所有权和借用的第一轮概念

这一章建议你先带着一个问题往下看：

为什么 Rust 这么喜欢 `enum + match` 这组组合？

因为它不只是“语法好看”，而是能把状态和分支写得非常明确。

## 3. 为什么光靠元组不够

元组很方便，但元组字段只有位置，没有名字：

```rust
fn main() {
    let user = ("Alice", 20, true);
    println!("{}", user.0);
}
```

这在小例子里可以，但一旦业务含义变复杂，读起来就会越来越别扭。

比如：

- `0` 到底是名字还是邮箱？
- `2` 到底是是否激活还是是否学生？

这时候更适合上 `struct`。

## 4. 结构体 `struct`

### 4.1 最基本写法

```rust
struct User {
    name: String,
    age: u32,
    is_active: bool,
}

fn main() {
    let user = User {
        name: String::from("Alice"),
        age: 20,
        is_active: true,
    };

    println!("{}", user.name);
}
```

这里的好处非常直接：

- 字段有名字
- 语义更清晰
- 后面扩展更自然

### 4.2 字段访问

```rust
println!("{}", user.age);
println!("{}", user.is_active);
```

字段访问用 `.`，直觉和很多语言类似。

### 4.3 可变结构体

如果你想修改字段，变量绑定本身要是可变的：

```rust
struct User {
    name: String,
    age: u32,
}

fn main() {
    let mut user = User {
        name: String::from("Alice"),
        age: 20,
    };

    user.age = 21;
    println!("{}", user.age);
}
```

### 4.4 结构体更新语法

Rust 还支持一种很常见的写法：

```rust
struct User {
    name: String,
    age: u32,
    is_active: bool,
}

fn main() {
    let user1 = User {
        name: String::from("Alice"),
        age: 20,
        is_active: true,
    };

    let user2 = User {
        name: String::from("Bob"),
        ..user1
    };

    println!("{}", user2.name);
}
```

这里先不深入细节，只先记住：它会把剩余字段从旧值搬过来，背后仍然和所有权规则有关。

## 5. 枚举 `enum`

如果说 `struct` 是“一个对象有很多字段”，那么 `enum` 更像：

一个值可能属于几种不同形态之一。

### 5.1 最简单的枚举

```rust
enum Direction {
    Up,
    Down,
    Left,
    Right,
}

fn main() {
    let dir = Direction::Up;
}
```

这里 `dir` 不会同时是四个方向，它只能是其中一种。

### 5.2 枚举为什么重要

很多语言也能表达“几种状态”，但 Rust 的 `enum` 特别强，是因为它的每个变体还能带数据。

例如：

```rust
enum Message {
    Quit,
    Write(String),
    Move { x: i32, y: i32 },
}
```

这里就非常强了：

- `Quit`：不带额外数据
- `Write(String)`：带一条字符串消息
- `Move { x, y }`：带具名坐标数据

这意味着一个值不仅有“类型名”，还有“当前是哪一种变体，以及这个变体携带了什么”。

## 6. `match` 和结构化拆解

这时 `match` 的威力就出来了。

```rust
enum Message {
    Quit,
    Write(String),
    Move { x: i32, y: i32 },
}

fn main() {
    let msg = Message::Write(String::from("hello"));

    match msg {
        Message::Quit => println!("quit"),
        Message::Write(text) => println!("text = {}", text),
        Message::Move { x, y } => println!("move to ({}, {})", x, y),
    }
}
```

这里不是简单比较值，而是在按“形态”拆值：

- 如果是 `Quit`
- 如果是 `Write(text)`
- 如果是 `Move { x, y }`

这就是模式匹配真正开始发挥价值的地方。

## 7. 结构体解构

结构体也可以在模式里拆开：

```rust
struct Point {
    x: i32,
    y: i32,
}

fn main() {
    let p = Point { x: 3, y: 7 };

    let Point { x, y } = p;
    println!("{} {}", x, y);
}
```

或者在 `match` 里：

```rust
struct Point {
    x: i32,
    y: i32,
}

fn main() {
    let p = Point { x: 0, y: 7 };

    match p {
        Point { x: 0, y } => println!("on y-axis: {}", y),
        Point { x, y: 0 } => println!("on x-axis: {}", x),
        Point { x, y } => println!("({}, {})", x, y),
    }
}
```

这说明模式匹配不只是 `enum` 专属，它是一整套“按形态拆值”的思路。

## 8. `if let` 的第一轮理解

有时候你不想写完整 `match`，只关心其中一种情况。

这时可以用 `if let`：

```rust
enum Direction {
    Up,
    Down,
}

fn main() {
    let dir = Direction::Up;

    if let Direction::Up = dir {
        println!("going up");
    }
}
```

你可以先把它理解成：

- “如果匹配这个模式，就执行这段代码”
- 它适合只关心一种分支时简化写法

## 9. `struct` 和 `enum` 分别什么时候用

### 更适合 `struct` 的场景

- 你要描述一个对象/实体
- 这个对象总是有这几项字段
- 重点是“有哪些属性”

例如：

- 用户
- 配置
- 坐标点
- 商品信息

### 更适合 `enum` 的场景

- 一个值只能处于几种状态之一
- 重点是“当前是哪一种”
- 不同情况可能带不同数据

例如：

- 请求状态
- 消息类型
- 操作结果
- 有值 / 没值

## 10. 和 JavaScript、Python、Java 的类比

### 和 JavaScript 类比

JavaScript 很多时候用普通对象加字符串字段来表示状态：

```js
const message = { type: "write", text: "hello" };
```

这很灵活，但也容易出现：

- `type` 拼错
- 某种状态缺字段
- 分支没处理全

Rust 的 `enum + match` 会更严格，也更安全。

### 和 Python 类比

Python 也能用字典或类表达这些结构，但“当前到底是哪一种状态、是否处理完整”很多时候靠约定。Rust 更偏向让类型系统参与保证。

### 和 Java 类比

Java 初学者更容易接受 `class` 对应 `struct` 的直觉，但 Rust 的 `enum` 往往比传统枚举更强，因为它能携带数据。这是一个非常值得重视的 Rust 特点。

## 11. 常见错误与如何调试

### 错误 1：该用 `struct` 的地方硬用元组

短期省事，长期可读性很差。

### 错误 2：以为 `enum` 只是“几个常量”

Rust 的 `enum` 不只是标签，还能带数据，这一点非常关键。

### 错误 3：`match` 没覆盖完整

Rust 提醒你“匹配不完整”不是在刁难你，而是在帮你把边界想清楚。

### 错误 4：匹配时把数据拆错

例如你以为某个变体带的是一个值，实际上带的是具名字段，这时先回头看 `enum` 定义本身。

## 14. 小结

这一章最关键的理解是：

1. `struct` 适合表示“一个对象有哪些字段”
2. `enum` 适合表示“一个值可能是几种形态之一”
3. Rust 的 `enum` 可以携带数据
4. `match` 的强大之处在于它能按结构拆值
5. `enum + match` 是 Rust 非常核心的一组能力
