# 迭代器与闭包入门

## 1. 这一章是干什么的

前面你已经用过：

- `for`
- `Vec<T>`
- `HashMap<K, V>`

这些足够你完成大量基础任务。

但 Rust 里还有两类非常核心、并且会反复出现的能力：

- 迭代器
- 闭包

这一章的目标是让你先建立第一轮实用直觉：

- 迭代器到底比普通循环多了什么
- 闭包和函数有什么关系
- 为什么它们经常一起出现

## 2. 学这一章前，你需要知道什么

你最好已经理解：

- `for` 循环
- `Vec<T>`
- 方法调用链

先带着一个总体印象往下看：

Rust 很多集合处理代码，并不只靠“手写循环 + 手动累加”，而更倾向于用迭代器方法表达“我要怎么处理这一串数据”。

## 3. 什么是迭代器

你可以先把迭代器理解成：

- 一种按顺序产出元素的机制

例如，对一个向量：

```rust
fn main() {
    let items = vec![1, 2, 3];

    for item in &items {
        println!("{}", item);
    }
}
```

你虽然写的是 `for`，但背后其实已经在用“逐个取元素”的思路。

而 Rust 的迭代器 API 则让你把这种“逐个处理”的逻辑写得更组合化。

## 4. 最小迭代器例子

```rust
fn main() {
    let items = vec![1, 2, 3];
    let iter = items.iter();

    for item in iter {
        println!("{}", item);
    }
}
```

这里先只抓住一点：

- `iter()` 会产生一个迭代器

它提供的是“依次查看元素”的能力。

## 5. `iter()`、`iter_mut()`、`into_iter()`

这是入门阶段最值得先分清的一组。

### `iter()`

- 产生不可变引用迭代
- 适合只读

```rust
for item in items.iter() {
    println!("{}", item);
}
```

### `iter_mut()`

- 产生可变引用迭代
- 适合修改元素

```rust
let mut items = vec![1, 2, 3];

for item in items.iter_mut() {
    *item += 1;
}
```

### `into_iter()`

- 消费原集合
- 把所有权交给迭代过程

```rust
let items = vec![1, 2, 3];

for item in items.into_iter() {
    println!("{}", item);
}
```

这里一个非常重要的直觉是：

- 这三者不只是“写法不同”
- 它们背后对应的是不同所有权方式

## 6. 什么是闭包

闭包可以先理解成：

- 可以放在变量里、传给别的函数的一小段匿名函数逻辑

最小例子：

```rust
fn main() {
    let add_one = |x| x + 1;
    println!("{}", add_one(5));
}
```

这里：

- `|x| x + 1`

就是闭包。

你现在可以先把它粗略理解成：

- 更轻量的临时函数

## 7. 闭包为什么和迭代器经常一起出现

因为迭代器经常要你回答：

- 每个元素怎么变换
- 哪些元素该保留
- 怎么把它们汇总

这些“小段逻辑”非常适合用闭包表达。

例如：

```rust
fn main() {
    let items = vec![1, 2, 3];

    let doubled: Vec<i32> = items.iter().map(|x| x * 2).collect();

    println!("{:?}", doubled);
}
```

这里：

- `map(...)` 在说：把每个元素变换一下
- `|x| x * 2` 就是变换规则

## 8. 常见迭代器方法：`map`

`map` 的直觉可以先记成：

- 把每个元素映射成另一个结果

```rust
fn main() {
    let items = vec![1, 2, 3];

    let result: Vec<i32> = items.iter().map(|x| x + 10).collect();
    println!("{:?}", result);
}
```

这里：

- 输入是 `1, 2, 3`
- 输出是 `11, 12, 13`

## 9. 常见迭代器方法：`filter`

`filter` 的直觉是：

- 只保留满足条件的元素

```rust
fn main() {
    let items = vec![1, 2, 3, 4, 5];

    let evens: Vec<&i32> = items.iter().filter(|x| **x % 2 == 0).collect();
    println!("{:?}", evens);
}
```

这个例子你先不用死磕 `**x` 的细节，先抓住：

- `filter` 在按条件筛元素

如果你一开始觉得闭包里类型细节有点绕，这是正常的。

## 10. 常见迭代器方法：`collect`

很多迭代器操作最后都不会立刻变成新的集合，而是先形成一个处理链。

`collect()` 的作用就是：

- 把迭代结果收集成某种具体容器

例如：

```rust
let result: Vec<i32> = items.iter().map(|x| x + 1).collect();
```

这里就是收集成 `Vec<i32>`。

## 11. 为什么有人喜欢迭代器风格

因为它经常能更直接表达“我要做什么”：

- 映射
- 过滤
- 收集

而不是把注意力都放在：

- 怎么建临时数组
- 怎么写循环变量
- 怎么手动 push

当然，入门阶段你不需要强迫自己把所有循环都改成迭代器链，但你要开始能读懂这种风格。

## 12. 一个稍微完整一点的例子

```rust
fn main() {
    let words = vec!["rust", "is", "fast"];

    let uppercased: Vec<String> = words
        .iter()
        .map(|word| word.to_uppercase())
        .collect();

    println!("{:?}", uppercased);
}
```

这段代码很典型：

1. 从集合拿迭代器
2. 对每项做变换
3. 收集成新集合

## 13. 闭包和函数的关系

你可以先这样理解：

### 普通函数更适合

- 逻辑可复用
- 逻辑较长
- 需要命名

### 闭包更适合

- 逻辑很短
- 只在本地用一次
- 和某个迭代器链天然绑在一起

## 14. 和 JavaScript、Python、Java 的类比

### 和 JavaScript 类比

如果你写过：

- `map`
- `filter`
- 箭头函数

那你会比较快适应 Rust 的迭代器 + 闭包风格。

### 和 Python 类比

如果你见过：

- 列表推导式
- `map`
- `filter`
- `lambda`

你也会比较容易接受“把变换逻辑作为参数传进去”的思路。

### 和 Java 类比

如果你接触过 Java Stream 和 lambda，会有一些熟悉感。但 Rust 的迭代器和所有权、借用耦合得更紧，所以你会更常注意 `iter()` / `iter_mut()` / `into_iter()` 的区别。

## 15. 常见错误与如何调试

### 错误 1：分不清 `iter()` 和 `into_iter()`

如果你发现集合后面不能再用了，先检查是不是被 `into_iter()` 消费掉了。

### 错误 2：在闭包里被类型细节绕住

先别急着抠每个符号，先理解：

- 这个闭包是在“变换”还是“筛选”

### 错误 3：一上来把所有普通循环全改成长链

这容易适得其反。入门阶段重点是：

- 能看懂
- 能写简单版本

### 错误 4：忘了 `collect()`

很多迭代器链如果你想变成具体集合，最后往往需要 `collect()`。

## 16. 小练习

1. 用 `map` 把一组整数都加 1。
2. 用 `filter` 筛出偶数。
3. 写一个闭包，把字符串变大写。
4. 对比 `iter()` 和 `into_iter()` 后，原集合还能不能继续用。

## 17. 挑战题

做一个“学习数据处理实验”，要求：

1. 准备一个学习主题列表
2. 用 `map` 把主题转成大写
3. 用 `filter` 筛出长度超过某个值的主题
4. 最后收集成新 `Vec`

额外要求：

1. 至少写两个闭包
2. 至少用一次 `collect()`
3. 至少自己对比一次 `iter()` 和 `into_iter()` 的差别

## 18. 小结

这一章最关键的理解是：

1. 迭代器是“按顺序产出元素”的机制
2. 闭包是轻量匿名函数
3. `iter()`、`iter_mut()`、`into_iter()` 对应不同所有权方式
4. `map`、`filter`、`collect` 是最常见的基础迭代器工具
5. Rust 的迭代器风格和所有权、借用是连在一起的
