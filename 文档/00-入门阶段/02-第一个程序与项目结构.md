# 第一个 Rust 程序与 Cargo 项目结构

## 1. 这一章是干什么的

这一章带你正式进入“一个 Rust 项目长什么样”。

你会学到：

- 怎么创建一个 Rust 项目
- `Cargo.toml` 是什么
- `src/main.rs` 是什么
- `cargo run` 背后大概做了什么
- 为什么 Rust 项目结构看起来简洁，但其实很讲究

如果上一章解决的是“工具装好了没有”，这一章解决的就是“项目怎么起步、代码从哪里开始看”。

## 2. 学这一章前，你需要知道什么

你最好已经知道：

- `cargo` 是 Rust 项目的主入口
- 命令行可以进入目录并执行命令

如果你学过其他语言，可以这样预热：

- JavaScript 里你可能先看 `package.json`
- Python 里你可能先看 `requirements.txt`、`pyproject.toml` 或入口脚本
- Java 里你可能先看 `pom.xml`、`build.gradle` 或 `src/main/...`

Rust 里，最重要的两个入口通常是：

- `Cargo.toml`
- `src/main.rs`

## 3. 核心概念

### 3.1 用 `cargo new` 创建项目

创建一个新项目最常见的方式：

```powershell
cargo new hello_rust
```

进入目录：

```powershell
cd hello_rust
```

运行项目：

```powershell
cargo run
```

如果一切正常，你会看到默认输出：

```text
Hello, world!
```

### 3.2 Cargo 默认帮你生成了什么

一个最基础的二进制项目通常会长这样：

```text
hello_rust/
├── Cargo.toml
└── src/
    └── main.rs
```

这套结构很小，但已经足够承载一个可运行程序。

### 3.3 `Cargo.toml` 是项目说明书

一个最小的 `Cargo.toml` 大概像这样：

```toml
[package]
name = "hello_rust"
version = "0.1.0"
edition = "2024"

[dependencies]
```

你现在先只需要认识这几个点：

- `[package]`：项目本身的信息
- `name`：项目名
- `version`：版本
- `edition`：Rust edition，用来决定语言版本语义
- `[dependencies]`：依赖列表

后面你加第三方库，通常就是在 `[dependencies]` 下面加内容。

### 3.4 `src/main.rs` 是二进制程序入口

默认的 `main.rs` 通常像这样：

```rust
fn main() {
    println!("Hello, world!");
}
```

这段代码先只看 3 个点：

1. `fn` 表示定义函数
2. `main` 是程序入口
3. `println!` 是打印到终端的宏

注意这里是 `println!`，后面有 `!`。这说明它不是普通函数，而是宏。你现在先记住“它是 Rust 里常见的输出方式”就够了。

## 4. `cargo run` 到底做了什么

很多新手会把 `cargo run` 理解成“直接运行脚本”，这不准确。

更接近的理解是：

1. Cargo 先检查项目配置
2. 调用编译流程
3. 生成可执行文件
4. 再运行这个可执行文件

所以 Rust 的常规开发体验不是“边解释边跑”，而是“编译后运行”。

这和 Python、JavaScript 的直觉不同，但和 Java 更接近。

## 5. 第一段改造练习

把默认程序改成这样：

```rust
fn main() {
    println!("Hello, Rust learner!");
    println!("I came from JavaScript, Python, and Java.");
}
```

然后执行：

```powershell
cargo run
```

这个练习虽然简单，但它在帮你建立一个最核心的反馈环：

- 修改源码
- 执行 `cargo run`
- 观察结果
- 再修改

以后你几乎每天都在重复这个循环。

## 6. 和 JavaScript、Python、Java 的类比

### 和 JavaScript 类比

如果你把 Rust 项目看成 Node.js 项目：

- `Cargo.toml` 有点像 `package.json`
- `cargo run` 有点像执行某个统一脚本入口

但重要区别是：Rust 最终是编译成二进制程序，不是默认依赖 JS 运行时去解释。

### 和 Python 类比

Python 里你可能会直接运行：

```powershell
python main.py
```

Rust 的新手常见等价直觉是“那我是不是应该直接跑 `main.rs`？”  
不完全对。Rust 项目更标准的做法是让 Cargo 负责工程流程。

### 和 Java 类比

Java 里你能感觉到“项目结构 + 构建工具 + 编译运行”是很重要的。Rust 也一样，但起步更轻，默认模板更小。

你可以粗略理解为：

- `Cargo.toml` 像一个更轻量的项目配置入口
- `src/main.rs` 像应用入口
- `cargo run` 帮你把构建和运行串起来

## 7. 为什么 Rust 项目结构值得尽早熟悉

因为后面你不只是会写一个 `main.rs`，还会逐步接触：

- 多模块拆分
- 库 crate 和二进制 crate
- 测试文件
- 示例代码
- 第三方依赖

如果你连最小项目结构都不熟，后面一多文件就容易乱。

所以现在不只是“把 Hello World 跑出来”，而是要开始建立目录感。

## 8. 一个稍微真实一点的例子

下面这个例子展示变量、字符串插值和基本输出风格：

```rust
fn main() {
    let name = "Rust";
    let goal = "build reliable software";

    println!("I am learning {} to {}.", name, goal);
}
```

如果你来自不同语言，可以这样看：

JavaScript 里你可能会写：

```js
const name = "Rust";
const goal = "build reliable software";
console.log(`I am learning ${name} to ${goal}.`);
```

Python 里你可能会写：

```python
name = "Rust"
goal = "build reliable software"
print(f"I am learning {name} to {goal}.")
```

Java 里你可能会写：

```java
String name = "Rust";
String goal = "build reliable software";
System.out.println("I am learning " + name + " to " + goal + ".");
```

Rust 在表面上不算难读，但后面它会在类型、所有权和借用上比这些语言严格得多。

## 9. 初学者常见错误

### 错误 1：在错误目录执行 `cargo run`

如果当前目录没有 `Cargo.toml`，Cargo 就不知道你要运行哪个项目。

### 错误 2：把 `Cargo.toml` 当成“可以随便写的普通文本”

它是项目配置文件，格式是 TOML，键值和节结构都要合法。

### 错误 3：只会复制模板，不知道各文件负责什么

你现在不需要会复杂配置，但至少要知道：

- `Cargo.toml` 管项目配置和依赖
- `src/main.rs` 管程序入口

## 10. 如何自己观察一个新 Rust 项目

以后你每次进入一个新项目，可以先按这个顺序看：

1. 先看有没有 `Cargo.toml`
2. 看项目名和依赖
3. 看 `src/main.rs` 或 `src/lib.rs`
4. 再看有没有额外模块和测试

这是一个很值得养成的阅读顺序。

## 11. 小练习

1. 自己新建一个项目，名字叫 `my_first_rust_app`。
2. 修改默认输出，让程序打印 3 行自我介绍。
3. 再额外加一行，写下你学 Rust 的原因。

## 12. 挑战题

做一个非常小的“个人卡片 CLI”，运行后输出类似内容：

```text
Name: your_name
Background: JavaScript / Python / Java
Current Goal: Learn Rust systematically
Next Milestone: Understand ownership
```

要求：

1. 用 `cargo new` 创建项目
2. 用 `cargo run` 运行
3. 自己修改输出内容，不要只照抄示例

如果你做完了，这已经不是单纯的 Hello World 了，而是你第一次真正“控制一个 Rust 项目输出什么”。

## 13. 小结

这一章你应该真正记住的是：

1. 新项目通常从 `cargo new` 开始
2. `Cargo.toml` 是项目配置入口
3. `src/main.rs` 是二进制程序默认入口
4. `cargo run` 不是简单“跑脚本”，而是“构建后运行”
