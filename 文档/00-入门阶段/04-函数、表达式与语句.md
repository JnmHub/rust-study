# 函数、表达式与语句

## 1. 这一章是干什么的

这一章会带你跨过 Rust 初学者的一个关键理解点：

- 函数怎么定义
- 参数和返回值怎么写
- 为什么 Rust 很强调“表达式”
- 为什么分号有时候只是格式，有时候会直接改变类型和返回值

如果你只把 Rust 当成“长得像 C/Java 的语言”，这一章很容易看漏真正重要的东西。Rust 的很多写法之所以顺，是因为它大量依赖表达式风格。

## 2. 学这一章前，你需要知道什么

你最好已经理解：

- `let` 可以绑定变量
- 基本类型大概有哪些
- `println!` 可以用来观察程序结果

另外，先带着这句话往下看：

在 Rust 里，不只是函数会产生值，很多代码块本身也会产生值。

这是后面理解 `if`、`match`、闭包、错误处理等内容的基础。

## 3. 核心概念

### 3.1 函数的基本写法

最基本的函数：

```rust
fn greet() {
    println!("Hello!");
}

fn main() {
    greet();
}
```

这里先注意 3 点：

1. 用 `fn` 定义函数
2. 函数名后面跟参数列表
3. 函数体写在 `{}` 里

### 3.2 带参数的函数

```rust
fn print_name(name: &str) {
    println!("Hello, {}!", name);
}

fn main() {
    print_name("Rust");
}
```

这里的 `name: &str` 表示：

- 参数名叫 `name`
- 参数类型是 `&str`

Rust 的参数必须写类型，这一点和 Python、JavaScript 很不一样，但和 Java 更接近。

### 3.3 带返回值的函数

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b
}

fn main() {
    let result = add(3, 4);
    println!("{}", result);
}
```

这里最关键的是：

```rust
a + b
```

这一行后面没有分号，所以它是一个表达式，值会作为函数返回值。

如果你写成：

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b;
}
```

就会出问题，因为加了分号后，这一行变成了语句，不再把值返回出去。

### 3.4 语句和表达式的区别

这是 Rust 学习中必须尽早理解的一个核心概念。

你可以先这样记：

- 语句：执行一个动作，不产生你要继续使用的值
- 表达式：会计算出一个值

例如：

```rust
let x = 5;
```

这是一条语句。

而：

```rust
5
```

这是一个表达式，因为它本身就是一个值。

再比如：

```rust
let y = {
    let x = 3;
    x + 1
};
```

这里整个代码块都是表达式，最终值是 `4`。

### 3.5 分号为什么重要

在 Rust 里，分号不仅仅是“语法结束符”，它有时直接改变代码意义。

看这个例子：

```rust
fn main() {
    let x = {
        let n = 10;
        n + 1
    };

    println!("{}", x);
}
```

这里 `x` 的值是 `11`。

但如果写成：

```rust
fn main() {
    let x = {
        let n = 10;
        n + 1;
    };
}
```

最后那个 `n + 1;` 变成了语句，代码块不再返回整数值，而是返回 `()`。

### 3.6 `()` 是什么

Rust 里有一个类型叫 unit，写作 `()`。

你可以先把它理解成：

- “没有有用返回值”
- 类似“空返回”

很多只负责打印、修改状态、执行动作的函数，返回的往往就是 `()`。

比如：

```rust
fn say_hi() {
    println!("Hi");
}
```

这个函数没有显式返回值，本质上返回的是 `()`。

### 3.7 显式 `return`

Rust 可以不用 `return`，但并不是不能用。

```rust
fn abs_like(n: i32) -> i32 {
    if n >= 0 {
        return n;
    }

    -n
}
```

什么时候 `return` 比较有用？

- 需要提前结束函数
- 分支比较复杂
- 你想让控制流更明显

但在简单函数里，Rust 社区更常见的是直接把最后一个表达式当返回值。

### 3.8 代码块本身也是表达式

这件事值得再强调一次，因为后面会反复用到：

```rust
fn main() {
    let level = 3;

    let description = if level > 5 {
        "advanced"
    } else {
        "beginner"
    };

    println!("{}", description);
}
```

这里的 `if` 不是单纯“执行逻辑”，它本身还是一个表达式，能产出值。

这和很多初学者印象里的“if 只是流程控制”不太一样。

## 4. 和 JavaScript、Python、Java 的类比

### 和 JavaScript 类比

JavaScript 里函数参数通常不强制写类型：

```js
function add(a, b) {
  return a + b;
}
```

Rust 则要求你更明确：

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b
}
```

同时，JavaScript 初学者对“表达式”通常有感觉，但不会像 Rust 这样频繁被分号和块表达式影响理解。

### 和 Python 类比

Python 的返回一般写得很直白：

```python
def add(a, b):
    return a + b
```

Rust 则允许“最后一个表达式直接返回”。

这会让代码很简洁，但也会让新手经常因为多写一个分号而报错。

### 和 Java 类比

Java 在函数定义和返回类型上会让你更熟悉 Rust 的形式感：

```java
static int add(int a, int b) {
    return a + b;
}
```

但 Java 一般不会像 Rust 这样频繁强调“块表达式返回值”。

Rust 的表达式风格比 Java 更强。

## 5. 常见错误与如何调试

### 错误 1：忘了写参数类型

```rust
fn add(a, b) {
    a + b
}
```

这在 Rust 里不行。函数参数类型必须明确。

### 错误 2：最后一行多写了分号

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b;
}
```

如果你看到报错大意是“expected `i32`, found `()`”，优先怀疑：

- 你本来想返回一个整数
- 结果最后一个表达式被分号变成了语句

### 错误 3：把打印和返回混在一起

```rust
fn add(a: i32, b: i32) -> i32 {
    println!("{}", a + b);
}
```

`println!` 主要是打印，不会帮你返回 `i32`。

### 错误 4：以为所有控制流都只是“执行动作”

Rust 的很多控制结构也会产生值，尤其是 `if` 和 `match`。如果你不建立这个意识，后面会很容易读不顺。

## 6. 小练习

1. 写一个 `say_name` 函数，接收你的名字并打印问候语。
2. 写一个 `square` 函数，接收整数并返回平方。
3. 写一个 `is_adult` 函数，接收年龄并返回 `true` 或 `false`。
4. 用代码块表达式计算一个值，例如：
   先定义局部变量，再返回处理后的结果。

## 7. 挑战题

做一个“学习进度判断器”，要求：

1. 写一个函数 `study_status(hours: i32) -> &'static str`
2. 如果学习时长小于 5，返回 `"just started"`
3. 如果学习时长在 5 到 10 之间，返回 `"keeping pace"`
4. 否则返回 `"serious learner"`

然后在 `main` 里调用它并打印结果。

额外要求：

1. 至少写一个带返回值函数
2. 至少用一次块表达式
3. 故意试一次“最后一行多写分号”，看报错长什么样

## 8. 小结

这一章最关键的理解是：

1. Rust 函数参数和返回值都很明确
2. 表达式会产出值，语句主要负责执行动作
3. 分号有时会直接改变返回值类型
4. 代码块、`if` 等结构都可能是表达式
