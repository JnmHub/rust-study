# 变量、可变性与基本类型

## 1. 这一章是干什么的

这一章解决的是 Rust 入门里最容易“看起来简单，实际上埋雷”的部分：

- 变量怎么声明
- 为什么默认不可变
- `let`、`mut`、类型标注分别在干什么
- 基本类型有哪些
- 元组、数组这些基础组合类型该怎么理解

如果你来自 JavaScript、Python、Java，表面上这些语法都不陌生。但 Rust 会在这些最基础的地方，提前把很多规则立住。

所以这一章的真正目标不是记住几个类型名，而是建立两个直觉：

1. Rust 默认更保守，它希望你明确表达“这个值会不会变”
2. Rust 的类型系统从一开始就在参与约束程序行为

## 2. 学这一章前，你需要知道什么

你最好已经知道：

- Rust 项目一般用 `cargo` 管理
- `main` 函数是程序入口
- `println!` 可以打印内容

如果你已经写过 JavaScript、Python、Java，可以先带着这几个对比问题往下看：

1. 为什么 Rust 不像 Python 那样随手改变量？
2. 为什么 Rust 不像 JavaScript 那样允许类型变化得很随意？
3. 为什么 Rust 的“变量绑定”更像一种带规则的声明，而不是单纯取个名字？

## 3. 核心概念

### 3.1 用 `let` 绑定变量

最基本的变量声明写法：

```rust
fn main() {
    let name = "Rust";
    let year = 2026;

    println!("{} {}", name, year);
}
```

这里先记住一句话：

`let` 在 Rust 里不是“随便定义一个以后再说的变量”，而是在做一次绑定。

也就是说，Rust 更强调“这个名字现在绑定到什么值、这个值之后能不能改”。

### 3.2 默认不可变

Rust 变量默认不可变：

```rust
fn main() {
    let score = 60;
    score = 80;
}
```

这段代码会报错，因为 `score` 默认不能重新赋值。

想让它可变，要显式写 `mut`：

```rust
fn main() {
    let mut score = 60;
    score = 80;

    println!("{}", score);
}
```

这就是 Rust 非常核心的风格之一：

- 默认安全
- 默认保守
- 默认不让你随便改
- 如果你确实要改，明确写出来

### 3.3 为什么默认不可变

这个设计不是为了“故意难受”，而是为了减少状态混乱。

如果一个值不会变，那么：

- 读代码的人更容易理解它
- 编译器更容易推断它
- 后面涉及借用、并发时也更安全

你可以先把这个规则理解成：

Rust 希望“变化是显式的，而不是隐含的”。

### 3.4 类型推断

Rust 经常可以自动推断类型：

```rust
fn main() {
    let count = 10;
    let pi = 3.14;
    let is_ready = true;
}
```

虽然你没写类型，但 Rust 会推断出：

- `count` 是整数类型
- `pi` 是浮点类型
- `is_ready` 是布尔类型

不过当上下文不够明确时，你也可以手动标注：

```rust
fn main() {
    let age: u32 = 18;
    let price: f64 = 19.99;
}
```

初学时可以这样理解：

- 能推断时，让 Rust 帮你推断
- 容易混淆时，自己标注类型

### 3.5 常量 `const`

变量和常量不是一回事：

```rust
const MAX_USERS: u32 = 100;

fn main() {
    println!("{}", MAX_USERS);
}
```

常量有几个特点：

- 必须写类型
- 不能用 `mut`
- 常量名通常全大写加下划线
- 它更适合放“程序运行期间不会变化的固定值”

你可以先把 `const` 想成“比不可变变量更强的一种固定声明”。

### 3.6 重新绑定与 shadowing

Rust 有个新手很容易混淆的点：变量不能随便改值，但可以重新绑定同名变量。

```rust
fn main() {
    let x = 5;
    let x = x + 1;
    let x = x * 2;

    println!("{}", x);
}
```

这里不是“修改同一个变量”，而是“用同一个名字绑定了新值”。

这叫 shadowing。

为什么这个设计有用？

因为有时候你想保留一个处理流程：

```rust
fn main() {
    let input = "42";
    let input = input.trim();
    let input: i32 = input.parse().expect("parse failed");

    println!("{}", input);
}
```

这比起强行用不同名字如 `input2`、`input3`，往往更自然。

### 3.7 标量类型

Rust 的基础标量类型你先抓住这几类：

1. 整数
2. 浮点数
3. 布尔
4. 字符

#### 整数

常见整数类型：

- `i32`：有符号 32 位整数
- `u32`：无符号 32 位整数
- `i64`、`u64`
- `usize`：常用于索引、长度等和平台相关的大小

示例：

```rust
fn main() {
    let a: i32 = -10;
    let b: u32 = 20;

    println!("{} {}", a, b);
}
```

#### 浮点数

常见是：

- `f32`
- `f64`

默认浮点字面量通常按 `f64` 处理。

```rust
fn main() {
    let x = 2.5;
    let y: f32 = 3.5;

    println!("{} {}", x, y);
}
```

#### 布尔

只有两个值：

- `true`
- `false`

```rust
fn main() {
    let is_done = false;
    println!("{}", is_done);
}
```

#### 字符

Rust 的字符是 `char`，用单引号：

```rust
fn main() {
    let letter = 'R';
    let heart = '❤';

    println!("{} {}", letter, heart);
}
```

注意：

- `char` 是单个字符
- `"R"` 是字符串
- `'R'` 才是字符

### 3.8 复合类型：元组

元组可以把多个不同类型的值打包在一起：

```rust
fn main() {
    let user = ("Alice", 20, true);

    println!("{}", user.0);
    println!("{}", user.1);
    println!("{}", user.2);
}
```

也可以解构：

```rust
fn main() {
    let (name, age, is_student) = ("Alice", 20, true);

    println!("{} {} {}", name, age, is_student);
}
```

元组适合：

- 临时返回多个值
- 小范围打包信息

但如果字段有明确业务含义，后面通常更应该用 `struct`。

### 3.9 复合类型：数组

数组的特点是：

- 长度固定
- 元素类型相同

```rust
fn main() {
    let scores = [90, 85, 78, 100];

    println!("{}", scores[0]);
    println!("{}", scores[3]);
}
```

也可以显式写类型：

```rust
fn main() {
    let numbers: [i32; 4] = [1, 2, 3, 4];
    println!("{}", numbers[2]);
}
```

这里 `[i32; 4]` 的意思是：

- 元素类型是 `i32`
- 长度是 4

Rust 会检查数组下标边界，越界会报错或在运行时 panic，而不是默默给你一个 undefined 风格的结果。

## 4. 和 JavaScript、Python、Java 的类比

### 和 JavaScript 类比

JavaScript 里你可能会这样写：

```js
let value = 10;
value = "hello";
```

这在 Rust 里不行，因为 Rust 不允许一个变量绑定先是整数、后又变成字符串。

Rust 更像是在说：

- 你可以灵活
- 但类型必须稳定

另外，JavaScript 的 `const` 更接近“这个绑定不能重新赋值”，而 Rust 的不可变 `let` 从默认就已经在强调这一点。

### 和 Python 类比

Python 里：

```python
x = 10
x = "hello"
```

这是很自然的。但 Rust 更希望你在一开始就把值的形态稳定下来。

这会让你一开始觉得不自由，但换来的是：

- 编译器能帮你发现更多问题
- 大型项目里更不容易乱

### 和 Java 类比

Java 初学者会较容易接受“变量有明确类型”这件事，但 Rust 在“默认不可变”上比 Java 更激进。

Java 里你常见的是：

```java
int score = 60;
score = 80;
```

Rust 则要求你显式承认这个值会变：

```rust
let mut score = 60;
score = 80;
```

## 5. 常见错误与如何调试

### 错误 1：忘了写 `mut`

示例：

```rust
fn main() {
    let count = 1;
    count = count + 1;
}
```

如果你看到报错大意是“cannot assign twice to immutable variable”，先别慌，它通常只是提醒你：

- 这个绑定默认不可变
- 你要么加 `mut`
- 要么换成 shadowing

### 错误 2：把字符和字符串混了

```rust
let a = 'A';
let b = "A";
```

这两个不是一回事：

- `'A'` 是 `char`
- `"A"` 更接近字符串切片字面量

### 错误 3：数组越界

```rust
fn main() {
    let arr = [1, 2, 3];
    println!("{}", arr[5]);
}
```

这不是一个“无所谓的小问题”，Rust 会把它当成真实错误看待。

### 错误 4：以为 shadowing 就是修改变量

不是。`let x = ...` 再写一次，是重新绑定，不是原地修改。

## 8. 小结

这一章最重要的结论是：

1. Rust 变量默认不可变
2. 想修改值，要显式写 `mut`
3. Rust 类型通常能推断，但你要理解类型一直在参与约束
4. 元组适合临时打包，数组适合固定长度同类型数据
5. shadowing 是重新绑定，不是原地修改
