# 入门阶段综合复盘与项目清单

## 1. 这一章是干什么的

这一章不是再教你新的语法点，而是帮你把整个入门阶段真正收口。

你已经走过了：

- 工具链
- 基础语法
- 所有权、借用、生命周期第一轮直觉
- 字符串、结构体、枚举、`Option`、`Result`
- 集合、模块、测试
- 泛型、trait、文件读写、CLI 小项目

如果现在不回头做一次系统复盘，你很容易出现一种错觉：

- “我好像都看过了，但又说不清哪些真的会了”

这一章的目标就是把这种模糊感压缩掉。

## 2. 你现在应该已经掌握的核心能力

到这个阶段，理想状态下你应该至少能做到下面这些事。

### 2.1 工具链层面

- 知道 `rustup`、`rustc`、`cargo` 分别干什么
- 会用 `cargo new`、`cargo run`、`cargo check`、`cargo test`
- 能读懂最小 Cargo 项目结构

### 2.2 语言基础层面

- 会写变量、函数、条件、循环
- 理解表达式和分号的差别
- 知道结构体和枚举分别适合什么场景

### 2.3 Rust 核心约束层面

- 理解所有权为什么存在
- 理解 move、借用、可变借用
- 能解释 `String` 和 `&str` 的差别
- 对生命周期至少有第一轮必要直觉

### 2.4 工程入门层面

- 会用 `Result` 处理失败
- 会用 `Vec` / `HashMap`
- 会写简单 `impl`
- 会拆基础模块
- 会补最基础测试

## 3. 如果你现在还会卡，最常见卡点在哪里

这很正常。入门阶段最常见的 6 个卡点通常是：

1. 所有权和 move 还不稳定
2. `String` / `&str` 经常混
3. `Option` / `Result` 还会本能想 `unwrap()`
4. `iter()` / `into_iter()` 直觉不稳定
5. 生命周期一看到就发怵
6. 一做小项目就容易把所有逻辑塞进 `main`

如果你有这些问题，不说明你学得差，只说明你正卡在 Rust 最有代表性的几个关口上。

## 5. 一个很实用的自测标准

你现在可以用下面这组问题检查自己。

如果你能用自己的话答出来，说明入门阶段已经不是“表面看过”。

1. 为什么 `String` 赋值常会 move，而 `i32` 不会？
2. `&String` 和 `&str` 在“只读参数”场景下为什么后者更常见？
3. `Option<T>` 和 `Result<T, E>` 分别在解决什么问题？
4. 为什么 `Vec::get()` 返回 `Option`？
5. `&self`、`&mut self`、`self` 分别意味着什么？
6. 为什么 `?` 能简化错误传播？
7. `iter()` 和 `into_iter()` 的核心差别是什么？
8. 为什么 Rust 要有生命周期？

## 6. 建议你在这个阶段完成的项目清单

下面这组项目，是非常适合入门阶段完成的。

### 第一组：非常小的热身项目

1. 个人信息卡片 CLI
2. 成绩统计器
3. 倒计时 / 学习时长记录器

目标：

- 熟悉变量、控制流、函数

### 第二组：基础工程项目

1. 文本统计 CLI
2. 文件预览器
3. 词频统计器

目标：

- 熟悉参数、文件、`Result`、集合、模块

### 第三组：结构与抽象训练

1. 学习计划管理器
2. 简单待办 CLI
3. 配置读取小工具

目标：

- 熟悉 `struct`、`enum`、`impl`、测试

## 7. 一个建议的项目升级顺序

同一个项目，不要只做一次。

更好的做法是：

1. 先写最小能跑版本
2. 再补错误处理
3. 再拆模块
4. 再补测试
5. 再重构掉明显难看的部分

这会让你真正体验到 Rust 学习里非常重要的一种能力：

- 不是只会写出第一版
- 而是会把第一版逐步写顺

## 8. 你什么时候可以认为自己“完成了入门阶段”

不是你把所有文档看完就算完成。

更现实的判断标准是：

1. 你能独立从零建一个 Cargo 小项目
2. 你能把逻辑拆成 2 到 3 个模块
3. 你能处理基本 `Result`
4. 你不会再把 `String` / `&str` 完全混掉
5. 你遇到 move / borrow 报错时，不会立刻完全无从下手
6. 你能给纯逻辑函数补基本测试

达到这些，你就已经从“纯新手”跨到了“真正开始进入 Rust 初级能力段”。

## 9. 入门阶段后，最自然的下一步是什么

下一阶段最值得继续强化的方向通常是：

1. 错误处理再进一层
2. 迭代器、闭包、trait、泛型真正写熟
3. 模块组织和项目结构更稳定
4. 开始做更完整的 CLI 或小型服务
5. 逐步接近并发、async、智能指针这些高级点

也就是说，下一步不是“重新学一遍基础”，而是开始把这些基础组合成更稳定的工程能力。

## 10. 建议你的复盘动作

如果你想把这一阶段学扎实，建议做下面 4 件事：

1. 重新做一遍你最不熟的 2 个章节练习
2. 完整做出 1 个综合 CLI 小项目
3. 对这个项目补至少 3 个测试
4. 用自己的话写一份“Rust 入门阶段我真正学会了什么”

这个动作非常值。

因为“写出来”比“以为自己会了”更接近真实掌握。

## 11. 你现在可以怎么继续用这个仓库

这个仓库接下来最自然的用法是：

1. 先继续补“初级阶段”文档
2. 同时开始做入门阶段项目

这样你会形成一个很健康的节奏：
