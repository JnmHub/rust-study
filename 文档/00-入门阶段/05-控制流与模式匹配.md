# 控制流与模式匹配

## 1. 这一章是干什么的

这一章会把 Rust 最常见的流程控制方式串起来：

- `if`
- `loop`
- `while`
- `for`
- `match`

如果说上一章是在建立“表达式”的感觉，这一章就是把这种感觉真正用在控制流里。尤其是 `if` 和 `match`，它们不只是“让程序走不同路线”，还经常直接参与产生值。

## 2. 学这一章前，你需要知道什么

你最好已经理解：

- 变量声明
- 带返回值函数
- 表达式和分号的区别

如果你还没完全熟，可以先记住一句话再往下看：

Rust 很多控制流不是单纯“流程工具”，而是“带类型约束的值选择工具”。

## 3. 核心概念

### 3.1 `if` 基础

Rust 的 `if` 写法很直接：

```rust
fn main() {
    let score = 85;

    if score >= 60 {
        println!("pass");
    } else {
        println!("fail");
    }
}
```

和 JavaScript、Python、Java 不同的一点是：Rust 的条件必须是布尔值。

这意味着你不能写这种模糊判断：

```rust
fn main() {
    let number = 1;

    if number {
        println!("yes");
    }
}
```

这会报错，因为 `number` 是整数，不是 `bool`。

Rust 要求你写清楚：

```rust
if number != 0 {
    println!("yes");
}
```

### 3.2 `if` 是表达式

这一点非常重要：

```rust
fn main() {
    let score = 85;

    let level = if score >= 90 {
        "excellent"
    } else if score >= 60 {
        "pass"
    } else {
        "fail"
    };

    println!("{}", level);
}
```

这里 `if` 表达式会返回一个值给 `level`。

但注意：各分支返回类型必须一致。

### 3.3 `loop`

`loop` 表示无限循环，除非你手动退出：

```rust
fn main() {
    let mut count = 0;

    loop {
        count += 1;

        if count == 3 {
            break;
        }
    }

    println!("{}", count);
}
```

`loop` 适合：

- 不确定循环次数
- 需要手动控制退出点
- 想配合 `break` 做更灵活的流程

### 3.4 `break` 也能带值

Rust 的 `loop` 有一个很有意思的特性：`break` 可以返回值。

```rust
fn main() {
    let mut count = 0;

    let result = loop {
        count += 1;

        if count == 5 {
            break count * 2;
        }
    };

    println!("{}", result);
}
```

这里 `result` 最终是 `10`。

这再次体现出 Rust 的风格：很多流程控制都不只是“执行”，还可以“产值”。

### 3.5 `while`

如果你有明确的循环条件，可以用 `while`：

```rust
fn main() {
    let mut n = 3;

    while n > 0 {
        println!("{}", n);
        n -= 1;
    }

    println!("done");
}
```

它的直觉和大多数语言差不多，但 Rust 不鼓励你写太多“纯手动下标循环”，后面会更常见 `for`。

### 3.6 `for`

Rust 里遍历集合时，`for` 往往是最常用、也最自然的方式：

```rust
fn main() {
    let items = ["Rust", "Python", "Java"];

    for item in items {
        println!("{}", item);
    }
}
```

也可以遍历范围：

```rust
fn main() {
    for n in 1..4 {
        println!("{}", n);
    }
}
```

这里 `1..4` 表示：

- 包含 1
- 不包含 4

如果想包含末尾值，可以写：

```rust
for n in 1..=4 {
    println!("{}", n);
}
```

### 3.7 `match`

`match` 是 Rust 的高频核心工具。

先看一个最小例子：

```rust
fn main() {
    let level = 2;

    match level {
        1 => println!("beginner"),
        2 => println!("intermediate"),
        3 => println!("advanced"),
        _ => println!("unknown"),
    }
}
```

这里的 `_` 可以理解成“其他情况”。

你可以把 `match` 暂时理解成：

- 比很多 `if else if` 更清晰的分支匹配工具
- 不只是比较值，还能逐步进入更强的模式匹配能力

### 3.8 `match` 也是表达式

`match` 不只是执行动作，也能返回值：

```rust
fn main() {
    let level = 2;

    let label = match level {
        1 => "beginner",
        2 => "intermediate",
        3 => "advanced",
        _ => "unknown",
    };

    println!("{}", label);
}
```

和 `if` 一样，各分支的结果类型需要一致。

### 3.9 模式匹配的第一层直觉

先不要把“模式匹配”想得太玄。

你现在先把它理解成：

- 不只是判断“等不等于某个值”
- 而是在问“这个值的形态是否符合某种模式”

例如：

```rust
fn main() {
    let point = (0, 7);

    match point {
        (0, y) => println!("on y-axis, y = {}", y),
        (x, 0) => println!("on x-axis, x = {}", x),
        (x, y) => println!("point = ({}, {})", x, y),
    }
}
```

这里就已经不是简单的“值等于几”，而是在根据结构去拆值。

## 4. 和 JavaScript、Python、Java 的类比

### 和 JavaScript 类比

JavaScript 的 `if` 常允许“真值/假值”风格：

```js
if (1) {
  console.log("yes");
}
```

Rust 不允许这种模糊写法，它要求布尔条件明确。

这是 Rust 的典型风格：不让你猜。

### 和 Python 类比

Python 也有很强的可读性，但在控制流里比 Rust 更宽松：

```python
if items:
    print("not empty")
```

Rust 会更明确，通常需要真正的布尔表达式。

### 和 Java 类比

Java 初学者会对 `if`、`while`、`for` 形式比较熟悉，但 `match` 往往会比传统 `switch` 更强，也更像“结构化分支”。

如果粗略对比：

- `match` 的使用地位往往比 Java 的 `switch` 更重要
- Rust 更鼓励你把分支写成可返回值的表达式

## 5. 常见错误与如何调试

### 错误 1：把整数直接当条件

```rust
if 1 {
    println!("hello");
}
```

Rust 不接受这种写法，条件必须是 `bool`。

### 错误 2：`if` 各分支返回不同类型

```rust
let value = if true { 1 } else { "hello" };
```

这会报错，因为一个分支是整数，一个分支是字符串。

### 错误 3：忘了 `match` 的兜底分支

如果你的匹配没有覆盖全部情况，Rust 会提醒你不完整。

这其实是好事，因为它在逼你把边界想清楚。

### 错误 4：用 `while` 硬写一切

Rust 里很多遍历更适合 `for`，尤其是对数组、切片、迭代器。过度依赖 `while` 往往代码更笨重，也更容易出错。

## 6. 小练习

1. 写一个程序，根据分数输出 `pass` 或 `fail`。
2. 用 `for` 打印 `1` 到 `5`。
3. 用 `while` 实现一个倒计时。
4. 用 `match` 根据星期数字输出星期名。
5. 用元组加 `match` 练习一次简单结构匹配。

## 7. 挑战题

做一个“学习阶段判断器”，要求：

1. 先定义一个整数 `hours`
2. 用 `if` 或 `match` 判断它属于哪个阶段
3. 输出类似：

```text
0..=5 -> warming up
6..=10 -> stable learning
11+ -> deep focus
```

额外要求：

1. 至少用一次 `for`
2. 至少用一次 `loop` + `break`
3. 至少用一次 `match` 返回值而不是只打印

## 8. 小结

这一章最关键的理解是：

1. Rust 条件必须是明确的布尔值
2. `if` 和 `match` 都经常作为表达式使用
3. `for` 是遍历时非常自然的入口
4. `match` 是 Rust 的核心分支工具，不只是“高级版 switch”
