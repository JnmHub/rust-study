# 引用与借用

## 1. 这一章是干什么的

上一章你已经知道：把 `String` 直接赋值给另一个变量，或者按值传进函数，往往会发生 move。

这一章要解决的就是紧跟着出现的问题：

- 那如果我只是想“用一下”这个值，不想拿走它，怎么办？
- 为什么有时候能同时借很多次，有时候又只能借一次？
- 什么是不可变引用，什么是可变引用？
- 为什么 Rust 会阻止“同时读写”同一份数据？

这就是引用和借用要回答的问题。

如果说所有权是在回答“谁拥有这个值”，那么借用是在回答“别人能不能暂时使用这个值，以及使用时要遵守什么规则”。

## 2. 学这一章前，你需要知道什么

你最好已经理解：

- move 是所有权转移
- `String` 按值传参可能让原变量失效
- 简单类型常是 `Copy`

这一章要先接受一个很关键的观念：

引用不是新的拥有者，它只是临时拿到“使用权”。

## 3. 为什么需要借用

先看一个最直接的问题：

```rust
fn print_text(text: String) {
    println!("{}", text);
}

fn main() {
    let s = String::from("hello");
    print_text(s);

    println!("{}", s);
}
```

这段代码会报错，因为 `s` 被 move 进函数了。

但很多时候，你其实只是想：

- 打印一下
- 看一下长度
- 做只读处理

你不是真的想把所有权交出去。

所以 Rust 提供了引用机制，让你可以“借来用一下”。

## 4. 什么是引用

最常见的引用写法是 `&`：

```rust
fn print_text(text: &String) {
    println!("{}", text);
}

fn main() {
    let s = String::from("hello");
    print_text(&s);

    println!("{}", s);
}
```

这里发生了两件事：

1. `print_text` 接收的是 `&String`，不是 `String`
2. 调用时传的是 `&s`，也就是对 `s` 的引用

这意味着：

- 函数只是借用了 `s`
- 所有权还在 `main` 里的 `s`
- 所以函数调用后，`s` 还能继续用

## 5. 借用到底是什么

你可以先把借用理解成：

- 不拿走值
- 暂时获得访问权限
- 原主人还在

所以引用和 move 的差别是：

- move：换主人
- borrow：临时借用，主人不变

## 6. 不可变引用

最常见的是不可变引用，也叫只读借用：

```rust
fn length_of(text: &String) -> usize {
    text.len()
}

fn main() {
    let s = String::from("hello");
    let len = length_of(&s);

    println!("len = {}", len);
    println!("s = {}", s);
}
```

这里 `length_of` 只是读取，不修改。

这类借用的关键特征是：

- 可以读
- 不能改

## 7. 可变引用

如果你想通过借用去修改值，就要用可变引用：

```rust
fn add_suffix(text: &mut String) {
    text.push_str(" world");
}

fn main() {
    let mut s = String::from("hello");
    add_suffix(&mut s);

    println!("{}", s);
}
```

这里要同时注意两件事：

1. 原变量本身必须是 `mut`
2. 借用时也要写 `&mut`

也就是说，“值本身可变” 和 “借用方式是可变借用” 这两个条件要同时满足。

## 8. Rust 借用规则的第一轮理解

这是这一章最核心的部分。

Rust 在同一时刻允许的借用方式大致可以先记成一句话：

- 要么多个不可变引用
- 要么一个可变引用

不能同时又读又写到会产生冲突的程度。

### 8.1 多个不可变引用

```rust
fn main() {
    let s = String::from("hello");

    let r1 = &s;
    let r2 = &s;

    println!("{} {}", r1, r2);
}
```

这没问题，因为大家都只是读。

### 8.2 一个可变引用

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &mut s;
    r1.push_str("!");

    println!("{}", r1);
}
```

这也没问题，因为当前只有一个“能改”的借用者。

### 8.3 为什么不能同时出现冲突借用

看这个例子：

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &s;
    let r2 = &mut s;

    println!("{}", r1);
    println!("{}", r2);
}
```

这会报错。

为什么？

因为 Rust 不希望同一时刻一边有人在只读地看它，一边另一个人可能改它。

你可以把它先理解成：

Rust 在编译期就阻止“读写打架”。

## 9. 借用规则背后的直觉

初学时不要只背规则，尽量理解它想防什么：

1. 防止数据在读取过程中被别的地方偷偷改掉
2. 防止多个地方同时修改同一份数据
3. 为后面的并发安全打基础

这就是为什么 Rust 的借用规则看起来有点严格，但其实很讲道理。

## 10. 悬垂引用是什么，Rust 为什么要阻止

看这个例子：

```rust
fn dangle() -> &String {
    let s = String::from("hello");
    &s
}
```

这在 Rust 里不行。

原因很直接：

- `s` 在函数结束时就被清理
- 你却想把它的引用返回出去
- 返回出去之后，这个引用会指向已经无效的东西

这就叫悬垂引用。

Rust 会在编译期直接拒绝这种情况。

这里你先不用急着学生命周期语法，只要先建立一个直觉：

引用不能比它指向的数据活得更久。

## 11. 什么时候该传值，什么时候该传引用

这是你开始写函数后会经常碰到的问题。

### 更适合传值的时候

- 你就是想把所有权交给函数
- 类型本身很轻量，常是 `Copy`
- 这个值后面本来就不打算继续用了

### 更适合传引用的时候

- 你只是想读
- 你不想失去原值所有权
- 数据比较大，不想复制

### 更适合传可变引用的时候

- 你想让函数帮你修改原值
- 但又不想把所有权交出去

## 12. 和 JavaScript、Python、Java 的类比

### 和 JavaScript 类比

JavaScript 里对象常常表现得像“默认共享引用”，但语言不会在编译期严厉管你什么时候读、什么时候写。

Rust 不一样。它不是只提供一个“能引用就行”的机制，而是把“怎么引用才安全”也一并规定了。

### 和 Python 类比

Python 里函数传参经常让人形成一种感觉：对象传进去后，函数里也能改，外面也能看，规则比较灵活。

Rust 则更明确：

- 借用是借用
- 修改必须显式声明
- 同时读写冲突要在编译期解决

### 和 Java 类比

Java 里对象变量更像引用语义，但编译器不会像 Rust 这样细致地追踪“当前有没有只读引用和可变引用冲突”。

Rust 的要求更高，但也正因为这样，它能更早发现很多状态问题。

## 13. 常见错误与如何调试

### 错误 1：函数参数写成 `String`，结果把值 move 走了

```rust
fn print_it(s: String) {
    println!("{}", s);
}
```

如果你只是想读，先想一想是不是应该改成：

```rust
fn print_it(s: &String) {
    println!("{}", s);
}
```

### 错误 2：想修改值，但忘了 `&mut`

```rust
fn update(s: &String) {
    s.push_str("!");
}
```

这会报错，因为 `&String` 是不可变引用。

### 错误 3：原变量不是 `mut`

```rust
let s = String::from("hello");
let r = &mut s;
```

这不行，因为你要创建可变引用，原值本身就必须允许修改。

### 错误 4：同时有不可变引用和可变引用

如果你看到报错大意是：

- cannot borrow as mutable
- because it is also borrowed as immutable

先别慌，优先检查：

1. 我是不是先创建了只读引用
2. 又在它还活着的时候创建了可变引用

## 14. 小练习

1. 写一个函数，接收 `&String`，打印字符串长度。
2. 写一个函数，接收 `&mut String`，在结尾追加 `"!"`。
3. 自己写一段代码，同时创建两个不可变引用，观察为什么合法。
4. 再自己写一段代码，同时创建一个不可变引用和一个可变引用，读懂报错。

## 15. 挑战题

做一个“小型文本处理器”，要求：

1. 定义一个 `String`
2. 写一个只读函数，打印它的长度
3. 写一个可变函数，给它追加一段后缀
4. 在 `main` 中先读，再改，再读

额外要求：

1. 故意写一次错误版本，让只读引用和可变引用冲突
2. 用你自己的话总结：为什么 Rust 不允许它通过

## 16. 小结

这一章最关键的理解是：

1. 引用不会拿走所有权
2. 借用就是临时使用值
3. `&T` 适合只读，`&mut T` 适合修改
4. Rust 借用规则可以先记成：多个只读，或者一个可写
5. Rust 会阻止悬垂引用和读写冲突
