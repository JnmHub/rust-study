# 入门阶段导读

## 1. 这一章是干什么的

入门阶段的目标很明确：先把你从“听说过 Rust”带到“我已经能自己跑起来一个小程序，并且开始理解 Rust 为什么和别的语言不一样”。

这一阶段不追求你立刻掌握所有权、借用、生命周期的全部细节，但会提前为它们铺路。因为 Rust 的很多语法表面看起来普通，背后的约束却不普通。

如果把学习过程类比成打游戏，那么这一阶段不是“刷满技能树”，而是先把地图打开、UI 看懂、基本操作练熟。

## 2. 学这一阶段前，你需要知道什么

你不需要有 Rust 基础。

你只需要对这些东西有一点感觉就够了：

- 变量和函数是什么
- `if`、`for`、`while` 是做什么的
- 命令行里运行程序是什么概念
- 你见过至少一种包管理工具

你学过 JavaScript、Python、Java，这对你是加分项。因为后面很多理解都可以借这些经验做桥接：

- JavaScript：帮助你理解“表达式感”和闭包
- Python：帮助你快速理解集合、脚本、工具链使用
- Java：帮助你理解静态类型、编译、泛型和工程化结构

## 3. 这一阶段到底学什么

入门阶段主要解决 4 类问题：

1. 工具问题
   你要知道 `rustup`、`rustc`、`cargo` 分别是什么，谁负责安装，谁负责编译，谁负责项目管理。
2. 代码外形问题
   你要看懂一个 Rust 文件最小长什么样，一个 Cargo 项目最小长什么样。
3. 基础语法问题
   你要掌握变量、函数、控制流、字符串、结构体、枚举这些最常用的基础语法。
4. 心智模型问题
   你要开始接受 Rust 的几个关键习惯：默认不可变、显式类型、错误处理有类型、值有所有者。

## 4. 这一阶段的学习顺序

建议你按下面顺序推进：

1. 先学环境安装与工具链
2. 再学第一个 Rust 程序与 Cargo
3. 然后进入变量、函数、控制流
4. 再进入字符串、结构体、枚举、模式匹配
5. 最后在真正讲所有权前，先建立“值会被移动”的直觉

不要一上来就死磕生命周期。

这和你学 JavaScript、Python 不一样。那两种语言你往往可以先写，后面再慢慢理解内部机制；Rust 则更像一门“写法受设计约束”的语言，你越早接受它的规则，后面越顺。

## 5. 你会遇到的第一个心理落差

很多初学者第一次看 Rust，会有两个连续反应：

1. “这语法不难啊，像 C/Java 一点，甚至比 C++ 简单。”
2. “怎么一写真实代码就开始报错，而且报错还不是语法错？”

这是正常的。

Rust 的难点不在基础语法，而在“值如何流动、谁拥有它、谁能借它、什么时候结束生命周期”。也就是说，难点更偏规则系统，而不是表面写法。

你可以先把 Rust 粗略理解成：

- 有 Java 一样的编译期类型约束
- 有 Python/JavaScript 不常强迫你面对的内存与状态约束
- 有比很多主流语言更严格的“先证明安全，再允许你写”的倾向

## 6. 和 JavaScript、Python、Java 的第一轮类比

### Rust 和 JavaScript

JavaScript 很多时候是“先跑起来再说”，哪怕运行时再报错也行。Rust 更像“先过编译器这一关，确认很多潜在问题不会发生，再让你运行”。

JavaScript 里这段代码可能先跑再说：

```js
let name = null;
console.log(name.toUpperCase());
```

Rust 更偏向在设计上减少这种“运行到一半才发现不对”的情况，所以它会逼你更早处理空值、错误和值的归属。

### Rust 和 Python

Python 写工具非常快，但很多约束是运行时才体现。Rust 的工具链也很强，但它更重视“编译时先把坑踩掉”。

可以把 Rust 想成：

- 没有 Python 那么自由
- 但更适合做长期维护、性能要求高、并发要求高的程序

### Rust 和 Java

Rust 和 Java 都是静态类型、编译型语言，所以你会比纯脚本语言更快适应它的“先定义清楚再运行”。

但 Rust 和 Java 的一个关键不同点是：

- Java 大量依赖 GC 管理内存
- Rust 不依赖 GC，而是通过所有权和借用在编译期管理资源

这就是为什么 Rust 会在初期更“拧巴”，但也正因为这样，它能在很多场景下同时拿到安全和性能。

## 7. 这一阶段最重要的三个目标

如果你只盯住 3 个目标，建议就是这 3 个：

1. 见到 Cargo 项目不慌，知道从哪里开始读。
2. 能写出并运行一个小 CLI 程序。
3. 遇到基础报错时，知道是“语法不对”“类型不对”还是“所有权/借用有问题”。

只要这 3 个目标站住，后面学所有权时你不会完全失速。

## 8. 常见误区

### 误区 1：先把所有语法点看完再写

不行。Rust 需要“看一点，写一点，再看一点”。只读不写会让你误以为自己懂了。

### 误区 2：编译报错说明我不适合学 Rust

不对。Rust 初学体验本来就高度依赖报错学习。你不是在避免报错，而是在训练自己读懂报错。

### 误区 3：我有 Java/Python/JS 基础，所以入门阶段可以跳过

不建议。你能跳过的是“什么是变量和函数”，但你不能跳过 Rust 对这些概念施加的新规则。

## 9. 这一阶段怎么学最快

推荐你每篇文档都按这个顺序处理：

1. 先通读概念，不急着记住每个细节
2. 把最小示例自己敲一遍
3. 自己改 2 到 3 个参数，故意触发一些报错
4. 做完小练习
5. 最后再做挑战题

学 Rust 时，主动制造一点小错误是有价值的，因为它能让你更快建立“编译器在保护什么”的感觉。

## 10. 小练习

1. 用你自己的话回答：`rustup`、`rustc`、`cargo` 分别负责什么。
2. 回想一下 Java、Python、JavaScript 的项目启动方式，先猜一猜为什么 Rust 会更强调 Cargo。
3. 先不查资料，写下你对“所有权”这个词的直觉理解，后面再回来对照。

## 11. 挑战题

做一个“学习日志”Markdown，文件名可以叫 `my-rust-notes.md`，至少写下这 4 个问题的初始答案：

1. 我为什么学 Rust？
2. 我觉得 Rust 最难的地方会是什么？
3. 我已有语言经验里，哪一门最能帮助我学 Rust？
4. 我希望 4 周后自己能写出什么样的小程序？

这不是形式主义。它的作用是让你在后面遇到挫折时，知道自己到底在往哪里走。

## 12. 小结

入门阶段的任务不是“掌握 Rust”，而是先把入口打通。

你现在最需要建立的是这几个直觉：

- Rust 是一门规则感很强的语言
- 它的难点主要不是表面语法，而是值和资源的管理模型
- 你必须尽早熟悉工具链和项目结构
- 编译报错不是阻碍，而是学习材料的一部分
