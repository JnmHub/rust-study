# 所有权、借用与错误处理热身题

## 1. 这一章是干什么的

这一组题的目标非常明确：

- 把 Rust 初学者最容易反复卡住的几个点，集中练扎实

主要是：

1. move
2. 借用
3. 可变借用
4. `Option`
5. `Result`
6. `String` 与 `&str`

如果这些点不稳，后面一进入项目题就会经常：

- 题目还没开始做，先被编译器拦住

## 2. 学这一章前，你需要知道什么

建议你已经阅读过：

- [所有权初识](../00-入门阶段/06-所有权初识.md)
- [引用与借用](../00-入门阶段/07-引用与借用.md)
- [String 与 str](../00-入门阶段/08-String与str.md)
- [Option 与 Result 入门](../00-入门阶段/10-Option与Result入门.md)

## 3. 这一组题怎么做

做这组题时，最重要的不是速度，而是：

1. 先写出你对数据流的判断
2. 再决定参数要传所有权还是传借用
3. 看到报错时，先翻译成人话
4. 不要第一反应就是 `clone`

这组热身题更像在训练：

- 你的 Rust 第一反应

## 4. 热身题 1：修复 move 报错

题目：

写一个函数 `print_and_count`，接收一个 `String`，打印它，再返回字符串长度。

然后在 `main` 里继续尝试使用这个字符串。

要求：

1. 先故意写出会 move 的版本
2. 再改成合理版本
3. 解释为什么原版不能继续用

提示：

1. 你要先判断函数到底需不需要拿走所有权
2. 很多场景传 `&str` 或 `&String` 就够了

## 5. 热身题 2：只允许一个可变借用

题目：

写一个函数，给 `Vec<i32>` 追加一个值，并在追加前后打印长度。

要求：

1. 先尝试写出“同时存在多个可变借用”的错误版本
2. 再修正为合法写法
3. 用自己的话解释：为什么 Rust 要限制这个行为

提示：

1. 把“读长度”和“修改内容”分开理解
2. 想一想借用的作用范围

## 6. 热身题 3：`String` 和 `&str` 选择题

题目：

实现一个函数 `is_rust_topic`，判断一段文本是否和 Rust 学习有关。

要求：

1. 函数参数设计成最合适的类型
2. 解释为什么不是另一个类型
3. 给出 2 个调用示例：一个传 `String`，一个传字符串字面量

提示：

1. 问自己：函数是否需要拥有这段文本
2. 如果只是读取，通常不用拿所有权

## 7. 热身题 4：`Option` 改写空值逻辑

题目：

有一个“查找学习分数”的逻辑，找不到时不要返回魔法值 `-1`，而是返回更合理的 Rust 风格结果。

要求：

1. 先写一个不推荐版本
2. 再改成 `Option`
3. 调用方必须显式处理找不到的情况

提示：

1. Rust 很强调“缺失”也要进类型系统
2. 调用方不能靠猜测返回值含义

## 8. 热身题 5：`Result` 改写错误路径

题目：

写一个函数，把字符串解析成分数，要求分数必须在 `0..=100` 范围内。

要求：

1. 非数字输入要报错
2. 超范围输入要报错
3. 错误信息要能区分是哪一类问题

提示：

1. 不要一开始就 `unwrap`
2. 想一想“解析错误”和“业务规则错误”不是同一种事

## 9. 热身题 6：避免无意义 clone

题目：

给定一个 `Vec<String>`，统计其中总字符数。

要求：

1. 先写一个用了 `.cloned()` 的版本
2. 再改成借用遍历版本
3. 解释为什么第二版更自然

提示：

1. 统计字符数不需要获得所有权
2. 看看能不能只读每个元素

## 10. 常见错误与修正方向

### 错误 1：一报 move 就 `clone`

先判断是否其实只需要借用。

### 错误 2：看不懂借用报错就乱改生命周期

入门热身题大多数都不是生命周期问题。

### 错误 3：把 `String` 当默认参数类型

读取型函数很多时候更适合 `&str`。

### 错误 4：用魔法值表示失败

Rust 更适合 `Option` / `Result`。

## 11. 小练习

1. 把热身题 1 的函数分别改写成接收 `String`、`&String`、`&str` 三个版本，比较调用体验。
2. 写一句话解释什么叫“借用作用域”。
3. 写一句话解释为什么 `Result` 比“返回布尔值表示成败”更有信息量。

## 12. 挑战题

自己设计一个“学习记录输入校验器”，要求：

1. 输入包含课程名和分数
2. 课程名不能为空
3. 分数必须可解析且范围合法
4. 返回 `Result`
5. 至少有 2 类可区分错误

额外要求：

1. 至少解释一次：为什么这里不该把所有输入都拿成 `String`
2. 至少解释一次：为什么错误分类会影响后续调用方写法

## 13. 小结

这一组热身题最重要的是建立三个本能：

1. 先想借用还是所有权
2. 先把报错翻译成人话
3. 先用类型表达缺失和错误，而不是用魔法值糊过去
