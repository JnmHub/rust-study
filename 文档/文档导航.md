## 设计思路

这个仓库不会把所有知识点塞进一篇长文，而是按阶段拆开：

- 一篇阶段导读，负责告诉你“这一阶段学什么、为什么学、怎么学”
- 多篇专题正文，负责讲清一个主题
- 每篇附带小练习和挑战题

## 当前阶段

目前 `00-入门阶段` 已经基本收口，`01-初级阶段` 已经接近收口，`02-高级阶段` 已经开始建立入口。

入门阶段的目标，是先建立正确直觉：

- Rust 工具链分别干什么
- 一个 Rust 项目最小长什么样
- `cargo` 为什么是你的主入口
- Rust 代码为什么看起来和 Java、Python、JavaScript 有点像，但行为又不同
- Rust 为什么会在所有权这里突然变“严格”
- Rust 为什么会把“空值”和“错误”也做进类型系统
- Rust 怎么从“会写几行代码”过渡到“能组织一个小项目”
- Rust 怎么开始进入可复用抽象、真实输入处理和完整小项目阶段
- Rust 怎么把入门阶段收束成可复盘、可继续扩展的工程学习路线

初级阶段的目标，则进一步升级为：

- 把所有权与借用真正放进参数、返回值、结构体、集合遍历里理解
- 把 trait、泛型、生命周期放到同一个抽象视角下看
- 开始搭更完整的多模块命令行项目
- 开始建立“入口、配置、I/O、分析、输出、测试”分层意识
