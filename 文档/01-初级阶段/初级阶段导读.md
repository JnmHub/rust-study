# 初级阶段导读

## 1. 这一章是干什么的

如果说入门阶段解决的是：

- Rust 能不能跑起来
- 基础语法能不能看懂
- 所有权、借用、生命周期有没有第一轮直觉

那么初级阶段解决的就是另一件更关键的事：

- 你能不能开始稳定地写 Rust

很多人学 Rust 卡住，不是因为完全看不懂概念，而是因为：

1. 单个知识点都见过
2. 一组合起来就开始乱
3. 一写真实一点的代码就被借用检查器拦住
4. 一做项目就不知道怎么拆模块、怎么传值、怎么组织错误

初级阶段的任务，就是把这些“看过但不会稳用”的内容，压成真正能落地的能力。

## 2. 学这一阶段前，你最好已经掌握什么

建议你至少已经完成入门阶段的大部分内容，并且具备下面这些基础：

- 会用 `cargo new`、`cargo run`、`cargo check`、`cargo test`
- 看得懂 `struct`、`enum`、`match`
- 知道 `String`、`&str`、`Vec<T>`、`HashMap<K, V>` 是什么
- 对 move、借用、可变借用有第一轮理解
- 见过 `Result<T, E>` 和 `?`

不要求你已经很熟。

初级阶段本来就是拿来“把不稳定的地方练稳定”的。

## 3. 这一阶段和入门阶段最大的区别

入门阶段更像“建立地图”。

初级阶段更像“开始真的走路”。

你会明显感觉到学习重心发生变化：

1. 从记语法，转向判断“这个值该归谁管”
2. 从会写单文件，转向会拆模块
3. 从知道有 trait、泛型、生命周期，转向知道它们为什么会一起出现
4. 从会跑程序，转向会组织一个更像样的小项目

也就是说，这一阶段的核心不是“多背几个关键字”，而是开始建立 Rust 的工程直觉。

## 4. 初级阶段最核心的 4 个坎

### 4.1 所有权和借用开始进入真实场景

前面你已经知道：

- move 是什么
- `&T` 和 `&mut T` 是什么

但真实代码里更常见的问题是：

- 参数到底该传所有权、共享借用，还是可变借用
- 返回值到底该返回拥有的数据，还是借来的引用
- 结构体字段该放 `String` 还是 `&str`
- 遍历集合时为什么一不小心就 move 掉了

### 4.2 抽象不再是“只会语法”

你会开始真正把：

- `impl`
- trait
- 泛型
- 生命周期

放到同一个问题里理解。

比如：

- 一个函数想支持多种类型，什么时候该用泛型
- 一个类型想声明“我具备某种能力”，什么时候该用 trait
- 一个函数想返回借来的值，什么时候需要生命周期

### 4.3 代码组织开始重要起来

初学时你把代码都塞进 `main.rs` 也能忍。

但到了初级阶段，你会越来越需要：

- 把参数解析单独拆出来
- 把纯逻辑单独拆出来
- 把错误处理边界划清楚
- 给核心逻辑补测试

### 4.4 开始学习“读报错并重构”

Rust 初级阶段非常重要的一项能力，不是一次写对，而是：

- 看懂报错
- 判断是哪一类约束冲突
- 改接口、改作用域、改数据流，而不是只想“怎么绕过去”

## 5. 这一阶段主要学什么

这第一批初级阶段文档会围绕 3 条主线展开：

1. 所有权与借用综合强化
2. trait、泛型、生命周期的组合应用
3. 第一个更完整的多模块命令行项目

你可以把它理解成一个很自然的升级链路：

1. 先把“值如何流动”练顺
2. 再把“抽象如何成立”讲清
3. 最后把它们放进一个真实项目结构里

## 6. 和 JavaScript、Python、Java 的类比

### 和 JavaScript 类比

JavaScript 里你经常不太需要显式想“这个值现在归谁”。对象可以到处传，很多问题留到运行时再说。

Rust 不一样。

Rust 强迫你在编码时就说清：

- 这个值是被拿走了
- 还是只是临时借来看一眼
- 还是要借来修改

### 和 Python 类比

Python 做脚本很快，因为对象模型和运行时都很灵活。

Rust 则更强调：

- 提前把边界讲清楚
- 提前把失败路径表达出来
- 提前把可变和不可变分开

这会降低“先写起来”的轻松感，但会提升“后面不乱掉”的稳定性。

### 和 Java 类比

Java 里你已经见过：

- 静态类型
- 接口
- 泛型
- 多文件工程

这会帮你很快适应 Rust 的工程化结构。

但 Rust 和 Java 的关键差别仍然是：

- Java 很多对象生命周期交给 GC
- Rust 则要求你更清楚地表达资源归属和借用关系

## 7. 这一阶段最应该盯住的能力目标

如果你只盯住几个最关键的目标，建议是：

1. 能比较稳定地判断函数参数该用 `T`、`&T` 还是 `&mut T`
2. 能解释一个常见借用报错大概在说什么
3. 能看懂带 trait 约束和泛型参数的函数签名
4. 能把一个小 CLI 项目拆成多个模块
5. 能给纯逻辑代码补基础测试

## 8. 这一阶段怎么学会比较快

建议你用这个顺序：

1. 先看概念讲解
2. 立刻把示例自己敲一遍
3. 主动改动参数类型，观察报错变化
4. 做小练习
5. 把练习收束到一个小项目里

初级阶段千万不要只看不写。

因为这时最大的难点已经不是“看懂”，而是“自己下手时会不会卡住”。

## 9. 一个很实用的自测问题清单

看完初级阶段前几篇后，你最好能回答下面这些问题：

1. 一个函数什么时候应该拿走 `String`，什么时候只接收 `&str`？
2. 为什么很多结构体字段更常直接放 `String`，而不是放 `&str`？
3. 泛型解决的是哪一类问题，trait 解决的又是哪一类问题？
4. 为什么有些返回引用的函数需要显式生命周期？
5. `main.rs` 和 `lib.rs` 各自更适合承担什么职责？

如果这些问题你能用自己的话说出来，说明你已经不只是“看过”。

## 10. 小练习

1. 回想你以前写 JavaScript、Python、Java 时，参数传递最少会考虑哪些事；再对比 Rust，多出的那一层约束是什么。
2. 打开你之前写的一个小 Rust 练习，把其中一个 `String` 参数尝试改成 `&str`，观察调用端和函数体需要一起调整哪些地方。
3. 写下你现在对 trait、泛型、生命周期三者关系的第一印象，等后面学完再回头修正。

## 11. 挑战题

把你入门阶段做过的一个小 CLI 项目重新设计一遍，但暂时不要写代码，只做设计：

1. 列出它会有哪些模块
2. 列出哪些函数应该拿所有权，哪些函数只借用
3. 列出哪里可能返回 `Result`
4. 列出哪些地方最适合补测试

这个挑战看起来简单，但非常有价值，因为它会逼你从“写代码”切换到“设计数据流和边界”。

## 12. 小结

初级阶段最重要的变化是：

1. 你开始从“知道概念”走向“能稳定组合这些概念”
2. 你要开始认真面对接口设计、数据流、模块边界和错误传播
3. 你会越来越少问“这个语法怎么写”，越来越多问“这个值该怎么流动才合理”
