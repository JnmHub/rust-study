# Rust 学习路线总览

## 1. 这份路线图解决什么问题

你已经接触过 JavaScript、Python、Java，这很有价值，因为你对以下东西并不陌生：

- 变量、函数、条件、循环
- 类、对象、模块、异常
- 集合、迭代、并发、工程化工具

但 Rust 的难点不在“有没有这些概念”，而在“这些概念被重新约束了”。

最典型的就是：

- 默认不可变
- 所有权必须明确
- 借用规则在编译期就会拦住很多问题
- 错误处理更强调类型而不是抛异常
- 泛型、trait、生命周期会和“内存安全”强绑定

所以这套学习路线的核心目标不是背语法，而是建立 Rust 的思维方式。

## 2. 学习原则

为了学得更快，这个仓库后续内容会按下面的原则组织：

1. 先理解模型，再背语法。
2. 先会读报错，再追求写得快。
3. 每个难点都尽量用最小例子解释。
4. 遇到抽象概念时，适当对比 JavaScript、Python、Java。
5. 每学完一块，必须配小练习和一个挑战题。

## 3. 阶段总览

整个路线分成四个阶段：

1. 入门
2. 初级
3. 高级
4. 精通

另外会单独维护一条挑战训练线，用来强化“看到题目就能自己下手”的能力。

## 4. 第一阶段：入门

目标：先把 Rust 跑起来，并建立最基本的语言直觉。

这一阶段会解决的问题：

- Rust 工具链怎么装，`rustup`、`cargo`、`rustc` 分别干什么
- Rust 程序最基本的结构是什么
- 变量、常量、函数、控制流、基本类型怎么写
- 为什么 Rust 默认不可变
- `String` 和 `&str` 到底是什么
- `struct`、`enum`、`match` 为什么是 Rust 的常用组合
- `Option` 和 `Result` 为什么比“空值 + 异常”更重要

重点主题：

1. 开发环境与 Cargo 基础
2. 变量、可变性、基本类型
3. 函数、表达式、语句
4. 条件、循环、模式匹配
5. 字符串与切片
6. 结构体与枚举
7. `Option` / `Result` 入门

这一阶段的里程碑：

- 能读懂一个小型 Rust CLI 程序
- 能独立写出 100 到 200 行以内的小练习
- 开始理解“值的所有者”这个概念，但还不要求完全吃透

建议挑战：

- 猜数字游戏
- 简单成绩统计器
- 一个迷你待办事项 CLI

## 5. 第二阶段：初级

目标：真正跨过 Rust 初学者最痛的坎，也就是所有权、借用和基础抽象能力。

这一阶段会解决的问题：

- 所有权为什么不是“语法怪癖”，而是 Rust 的核心设计
- 引用、可变引用、借用规则到底如何避免悬垂引用和数据竞争
- 生命周期到底是什么，它什么时候需要显式写出来
- `Vec<T>`、`HashMap<K, V>`、切片、迭代器应该怎么配合
- `impl`、`trait`、泛型怎么构建可复用代码
- 错误处理如何从 `unwrap` 走向更可维护的写法
- 模块、crate、包管理如何组织项目

重点主题：

1. 所有权与 move
2. 借用与引用
3. 生命周期入门
4. 集合类型与字符串处理
5. 方法、关联函数、`impl`
6. trait 与泛型
7. 模块系统与可见性
8. 测试基础
9. 文件读写与命令行参数

这一阶段的里程碑：

- 能自己定位大多数初级借用报错
- 能写出结构清晰的小型命令行工具
- 能开始把 JavaScript/Python 的“灵活写法”转换成 Rust 的“显式写法”

建议挑战：

- 词频统计器
- 配置文件读取器
- 简易日志分析器
- 文本搜索工具

## 6. 第三阶段：高级

目标：从“会写 Rust”走向“能写不别扭、能处理真实复杂度的 Rust”。

这一阶段会解决的问题：

- 智能指针到底解决什么问题
- `Box<T>`、`Rc<T>`、`Arc<T>`、`RefCell<T>`、`Mutex<T>` 应该怎么选
- trait object 和泛型静态分发的差异是什么
- 多线程并发在 Rust 里怎么做
- `Send`、`Sync` 为什么重要
- `async/await` 和同步代码的思维差异是什么
- 如何设计更像工程代码的错误类型与模块边界
- 常见生态库应该怎么上手

重点主题：

1. 智能指针体系
2. 内部可变性
3. trait object 与动态分发
4. 多线程、channel、共享状态
5. `Send` / `Sync`
6. async/await 基础
7. 常见生态库：`serde`、`tokio`、`clap` 等
8. 工程化错误处理
9. 项目结构设计

这一阶段的里程碑：

- 能读懂中等复杂度的开源 Rust 项目
- 能写出具备分层结构的 CLI 或小型服务
- 能解释“为什么这段代码需要 `Arc<Mutex<T>>` 而不是别的”

建议挑战：

- 并发任务调度器
- 异步文件抓取器
- 简单 HTTP API 服务
- 支持配置与日志的命令行工具

## 7. 第四阶段：精通

目标：进入 Rust 的深水区，建立性能、抽象、边界和工程权衡能力。

这一阶段会解决的问题：

- 生命周期为什么在复杂 API 设计里会重新变难
- 如何写出零成本抽象，而不是“看起来泛型很多”
- 宏、unsafe、FFI、Pin、self-referential constraints 的边界在哪里
- 如何做性能分析、内存分析和基准测试
- 如何设计和发布一个质量合格的 crate
- 如何阅读标准库或成熟框架源码

重点主题：

1. 生命周期进阶
2. 高级 trait 技巧
3. 宏系统入门到实战
4. `unsafe` 的边界与原则
5. FFI 与跨语言交互
6. 性能优化与 profiling
7. 基准测试与可观测性
8. crate 设计、文档与发布
9. 阅读源码的方法

这一阶段的里程碑：

- 能设计比较稳定的公共 API
- 能分析性能瓶颈而不是盲目优化
- 能阅读和拆解优秀 Rust 项目的核心模块

建议挑战：

- 自己封装一个小型工具 crate
- 写一个简化版 key-value store
- 做一个 Rust 调用 C 或被其他语言调用的小示例
- 对现有项目做一次真实性能分析

## 8. 挑战训练线

除了主线教程，后续会单独安排一条训练线，重点不是“再讲一次”，而是让你练：

1. 看到需求后如何拆题
2. 看到编译报错后如何定位
3. 看到所有权冲突后如何重构
4. 看到并发场景后如何选数据结构
5. 看到库文档后如何提炼最小可用写法

训练题会分成三档：

- 热身题：巩固单一知识点
- 综合题：组合多个知识点
- 挑战题：更接近真实项目

## 9. 推荐学习节奏

如果你希望“效率高但不容易崩”，建议用这个节奏：

- 每次学习 45 到 90 分钟
- 每章分三步：先看讲解，再抄最小例子，再独立做练习
- 每周至少做一次“从零写一个小程序”的整合训练
- 每学完一个阶段，回头重写前面的一个小项目

## 11. 当前结论

这个仓库当前已经比较明确地形成了主线：

- 仓库会以 Markdown 教程为主
- 学习主线分为 入门 / 初级 / 高级 / 精通
- 每个阶段都会搭配练习与挑战
- 难点会尽量用 JavaScript、Python、Java 做类比解释

接下来最自然的推进方式是：

1. 把初级阶段继续补完整
2. 用更多多模块项目把抽象和工程结构练稳
3. 在初级阶段后半段逐步接近更真实的错误处理、迭代器组合和中型 CLI 设计
